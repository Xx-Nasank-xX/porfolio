SAE 1.03

Durant cette SAE, nous devions créer une documentation technique et utilisateur automatique pour un entreprise et ses fichiers C commentés. Pour cela nous avons utilisé Docker, et diverses langages de programmation comme du bash, du markdown,.. Cette SAE à débuté avec 3 séances pour apprendre l'utilisation de Docker, la première est la présentation\tuto de Docker, la deuxieme est la façon de renvoyer une documentation et la derniere partie est un programme bash qui permet de faire cette documentation technique automatiquement.

Les Etapes


  • 1ere Etape
    Nous avons commencé par la crétion de 3 fichiers C commentés.

  • 2eme Etape
    Ensuite nous avons dû définir la mise en page de la documentation technique avec un fichier html et un fichier en markdown.

  • 3eme Etape
    Après nous avons dû définir la mise en page de la documentation utilisateur avec un fichier html et un fichier en markdown.

  • 4eme Etape
    Ensuite nous avons créer un volume Docker appellé sae203.

  • 5eme Etape
    Pour continuer ce projet, nous avons créer un script php pour avoir la mise en page des documentations définis au-dessus.

  • 6eme Etape
    Pour finir le projet, nous avons créer un script bash pour automatiser et rélier tous les scripts

Compétences utilisées lors de ce projet


  • Compétence 1 : développer une application simple
    Durant le projet, j'ai développé un script bash permettant de automatiser le processus.

  • Compétence 3 : Administrer un réseau
    Nous avons du gérer la création et la suppression d'un volume Docker pour la création de la documentation

  • Compétence 6 : Travailer en équipe
    Nous avons travaillé en groupe de 4 pour réaliser ce projet.
